介绍

基于内存的key-value结构数据库，非关系型数据库

而mysql是磁盘存储，数据结构是二维表的形式

- 基于内存存储，读写性能高
- 适合存储热点数据（短时间内会被多次访问）（热点商品、资讯、新闻等）
- 数据结构不固定
- 对一致性，安全性要求不高
SQL NoSQL
1、结构化（Structured)
-对表字段进行约束，之后表的结构就固定了		 1、非结构化
-对表字段没有严格约束，且有多种存储结构
2、关联的（Relational)
-数据库会维护表之间的关系		 2、无关联的
-不会维护表之间的关系，需要程序员自己设计关联
3、SQL查询
-语法统一且复杂		 3、非SQL
-语法简单
4、ACID(事务） 4、BASE(基本无事务）

常用数据类型

key-value结构

key是字符串类型

value有五种常用的数据类型：

        + 字符串(string)

string类型是二进制安全的。意思是redis的string可以包含任何数据。比如jpg图片或者序列化的对象 。

底层是字节数组，int，float,string底层是一样的

string类型是Redis最基本的数据类型，一个键最大能存储512MB。

        + 哈希(hash)  也叫散列，类似于Java的HashMap, 适合存储对象
        + 列表(list)  按照插入顺序排序，可以有重复元素，类似于Java中的LinkedList。<font style="color:rgb(51, 51, 51);">你可以添加一个元素导列表的头部（左边）或者尾部（右边）。</font>
        + 集合(set)  无序集合，没有重复元素，类似于Java的HashSet  <font style="color:rgb(51, 51, 51);">集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是O(1)。</font>
        + 有序集合(sorted set / zset)  集合中每个元素关联一个分数（score)，根据分数升序排序，没有重复集合

zset 和 set 一样也是string类型元素的集合,且不允许重复的成员。

不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。

zset的成员是唯一的,但分数(score)却可以重复。

通用命令

KEYS 查看所有符合模版的key 例如KETS * 但是不建议在生产环境设备上使用，为单线程的，会阻塞

DEL key [key …] 会返回删除的个数

EXISTS key [key…] 判断是否存在，1是存在

EXPIRE 给一个key设置有效期，有效期到期时，该key会被自动删除

TTL 查看一个key的剩余有效期，若为-1，则为永久有效（在创建的时候不设置有效时间，就会被设置为永久有效）

String类型将对象序列化为JSON字符串存储，后续如果需要修改对象的某个字段的时候很不方便

List类型

与Java的LinkedList类似，可以看做是一个双向链表结构，既可以正向检索也可以反向检索

有序，元素可以重复，插入和删除快，查询速度一般

对于要求顺序的数据可以使用这种结构

可以用List去模拟栈，队列，阻塞队列

Set类型

类似于Java的HashSet，可以看做是一个value为null的HashMap

无序，元素不可重复，查找快，支持交集并集差集等功能

SortedSet

是一个有序的set集合，与Java的TreeSet有些相似，但是底层是每一个元素都带有一个score属性，可以基于score属性对元素排序，底层的实现是一个跳表（SkipList)加hash表

可排序，元素不可重复，查询速度快，经常被用来实现排行榜的功能

注意：所有的排名默认为升序的，如果要降序则是在命令Z后面添加REV即可

Redis客户端

1、**jedis ** 与Redis命令为方法名称，学习成本低，但是线程不安全，多线程环境下需要用连接池来使用

2、lettuce 基于Netty实现的，支持同步，异步和响应式编程方式，并且是线程安全的。支持Redis的哨兵模式，集群模式和管道模式（SpringBoot默认集合的这个客户端）

3、Redission 是一个基于Redis实现的分布式，可伸缩的Java数据结构集合。包含了诸如map，queue，lock，semaphore，atomiclong等强化功能
