数组理论

数组是存放在连续内存空间上的相同类型数据的集合。

数组可以方便的通过下标索引的方式获取到下标对应的数据。

    • 数组下标都是从0开始的。
    • 数组内存空间的地址是连续的

正因为如此，对数组进行删除操作的时候，需要注意数组中元素的连续性，所以在增删操作的时候，难免要影响别的元素位置

像Java是没有指针的，同时也不对程序员暴露其元素的地址，寻址操作完全交给虚拟机。

    • 一旦创建，长度就无法更改
    • 是一个对象，有相应的属性和方法

704. 二分查找 - 力扣（LeetCode）

题目描述：

给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target ，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

  • 二分查找的要点：

    对于区间[left,right],target会处于这个区间之内，所以

        • while中写的是left <= right，因为left == right是有意义的，可能就是我们要找的target
        • 在缩小范围的时候，每个都是为mid + 1 / mid - 1，因为为双闭区间，right和left上面的元素是取得到的，可以说明这个元素一定不为target，所以缩小范围的时候需要除去这个元素
        • 并且进行条件判断的时候都用的是< >

    对于区间[left,right)，会与上一种不同：

        • while中写的是left < right，因为left == right是没有意义的，因为右边为开区间，可能取到无效元素
        • 对于区间的更新，当查找元素比中间元素大的时候，right = mid,因为目标元素在左区间，而右边为开区间，mid是取不到的，所以更新的时候，right移动到mid的位置。而当查找元素比中间元素小的时候，说明元素在右区间，更新为left = mid + 1,因为左区间是闭区间，mid是取得到的，所以下次考虑的时候不能包括mid元素。

    • 时间复杂度：O(log n)

    • 空间复杂度：O(1)

27. 移除元素 - 力扣（LeetCode）

题目描述：

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并原地修改输入数组。元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

示例 1: 给定 nums = [3,2,2,3], val = 3, 函数应该返回新的长度 2, 并且 nums 中的前两个元素均为 2。 你不需要考虑数组中超出新长度后面的元素。

示例 2: 给定 nums = [0,1,2,2,3,0,4,2], val = 2, 函数应该返回新的长度 5, 并且 nums 中的前五个元素为 0, 1, 3, 0, 4。

你不需要考虑数组中超出新长度后面的元素。

解决要点：

题目要求原地处理数组，但是我们又需要移除与val相同的元素，并且用后面的元素补齐空位，这看似很难进行操作，无法符合题目原地处理的要求。

想法是通过双指针解决，因为最后要求与val值不同的元素数量为K,数组的前K个元素都要是与val不同的元素，其他的不用管，所以可以用一个指针去遍历数组，去找与val不同的元素，而另外一个指针则是记录存放元素的位置，这样当第一个指针遍历完，会把与val不同的元素根据第二个指针都移动到前面

977. 有序数组的平方 - 力扣（LeetCode）

题目描述：

给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

示例 1：

  • 输入：nums = [-4,-1,0,3,10]
  • 输出：[0,1,9,16,100]
  • 解释：平方后，数组变为 [16,1,0,9,100]，排序后，数组变为 [0,1,9,16,100]

示例 2：

  • 输入：nums = [-7,-3,2,3,11]
  • 输出：[4,9,9,49,121]

解题要点：

题目要求为对数组每一个元素进行平方，然后返回的结果要求是递增顺序的。

需要注意的一点是原数组，可能会同时有正数和负数，而负数的平方可能还比正数大，所以暴力的做法是对每一个元素进行平方，再进行排序。

另外一种做法是双指针的做法。在原数组的左和右俩个指针，用来找平方之后的最大元素，因为数组本身就是有序的，所以最大的，要么在左边，要么在右边。然后开辟一个新数组，用一个反向指针，依次存入相比之下，较大的元素。最后直接返回新数组即可，因为是每次将最大的元素反向存入，所以结果是有序的。
